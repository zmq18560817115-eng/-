# 膝悦 KneeJoy — 云端部署 + 硬件联调（小白教程）

> 目标：**软件跑在免费云服务器上**，测试设备（手机/平板）打开网页就能用；**硬件（ESP32 + 推杆）** 能收到真实控制指令，而不是纯虚拟演示。

---

## 一、先搞懂：三个东西各干什么

```
┌─────────────────┐      HTTPS       ┌──────────────────────┐      Wi-Fi/MQTT      ┌─────────────┐
│  测试设备        │  ──────────────►  │  云服务器（免费）      │  ◄──────────────►  │  ESP32+推杆  │
│  手机/平板浏览器  │   登录、开理疗、   │  Web页面 + API接口    │   拉力/温度/停止    │  真实硬件    │
│                 │   看记录          │  （Render 等）         │                     │             │
└─────────────────┘                   └──────────────────────┘                      └─────────────┘
```

| 部分 | 放哪里 | 现在状态 | 你要做什么 |
|------|--------|----------|------------|
| **Web 界面** | 云服务器 | ✅ 已有 | 按下面步骤部署 |
| **API 后端** | 云服务器 | ✅ 已有 | 同上，和 Web 一起打包 |
| **硬件固件** | ESP32 里 | ❌ 还没有 | 面包板阶段写固件 |
| **硬件驱动** | `src/hardware/` | ⚠️ 目前是 mock | 后续换成 Wi-Fi/BLE 真驱动 |

**重要：** 云服务器**不能直接插 USB 控制推杆**。必须是：

- **方案 A（推荐毕设）：** 测试设备浏览器 → 云 API → **ESP32 连 Wi-Fi** 收指令 → 推杆  
- **方案 B：** 测试设备浏览器用 **Web 蓝牙** 直连 ESP32（人要在设备旁边，云只存数据）

毕设答辩、远程演示，**优先方案 A**。

## 二、第一阶段：免费部署到 Render（约 30 分钟）

### 为什么选 Render？

| 平台 | 免费？ | 能跑你的 Docker？ | 说明 |
|------|--------|-------------------|------|
| **Render** | ✅ 有免费层 | ✅ 有 `Dockerfile` | **推荐**，一条命令部署 Web+API |
| Netlify | ✅ | ❌ 只能放静态页 | 不能跑 Express 后端 |
| 自己 Mac | 免费 | ✅ | 不能 24 小时开着，不算「部署好了」 |

免费层限制（要知道）：

- **15 分钟没人访问会休眠**，第一次打开要等 ~30 秒唤醒  
- 免费实例 **重启后 JSON 数据库可能清空**（演示够用；正式用要换 PostgreSQL）  
- 每月约 **750 小时** 免费运行时间

### 步骤 3：创建 Web 服务（两种方式任选）

#### 方式 A：Blueprint 一键（最简单）

1. Render 控制台 → **New +** → **Blueprint**  
2. 连接你的 GitHub 仓库  
3. Render 会读取根目录 `render.yaml`，直接 **Apply**  
4. 等 5～10 分钟构建完成  

#### 方式 B：手动创建

1. **New +** → **Web Service**  
2. 选你的 GitHub 仓库  
3. 填写：

| 配置项 | 填什么 |
|--------|--------|
| Name | `kneejoy`（随意） |
| Region | Singapore 或 Oregon（离中国近选 Singapore） |
| Branch | `main` |
| Runtime | **Docker** |
| Instance type | **Free** |
| Health Check Path | `/api/v1/health` |

4. 环境变量（Environment）：

| Key | Value |
|-----|-------|
| `NODE_ENV` | `production` |
| `API_PORT` | `8080` |

5. 点 **Create Web Service**，等待 Deploy 成功

## 三、第二阶段：从「虚拟演示」到「真硬件」

### 当前软件在干什么（虚拟部分）

- 点「连接设备」→ 走 `src/hardware/mockAdapter.ts`，**假装**连上了  
- 点「开始理疗」→ 只在网页和后端 JSON 里改数字，**推杆不会动**  

### 你要补的两块

#### 块 1：ESP32 固件（硬件同学 / 你本人）

在 ESP32 上实现：

1. 连家里或实验室 **Wi-Fi**  
2. 定期访问云 API，或订阅 **MQTT** 消息  
3. 收到指令例如 `@STxCMD: L_F=10, R_F=10, TEMP=40, VIB=2` → 驱动推杆/加热/震动  
4. **安全逻辑写在 ESP32 里**：安全夹未插紧 → 拒绝启动；超力 → 立刻停  

#### 块 2：软件里的硬件驱动（`src/hardware/`）

把 `mockAdapter.ts` 换成真实适配器，例如：

- **Wi-Fi 方案：** 网页点「开始」→ 调 API → API 通知 ESP32（或 ESP32 轮询 `/device/command`）  
- **蓝牙方案：** 仅 Chrome 安卓支持 Web Bluetooth；iPhone Safari 支持很差  

**毕设推荐 Wi-Fi 方案**，和「软件在云上」一致。

---

### 推荐联调顺序（面包板阶段）

```
第 1 周   ESP32 串口打印：收到 START / STOP 就行
第 2 周   ESP32 连 Wi-Fi，能访问你的 Render health 地址
第 3 周   ESP32 调 Render API 上报 telemetry（项目里已有 POST /patients/me/device/telemetry）
第 4 周   网页点「开始理疗」→ 后端写指令 → ESP32 拉取并驱动推杆
```

## 四、硬件和云端怎么「说话」（简化协议示例）

### 云端 → ESP32（下发指令）

可新增一个简单接口（后续开发），例如：

```
GET https://你的域名.onrender.com/api/v1/device/esp32/commands?device_id=KJ001
```

返回：

```json
{
  "cmd": "START",
  "left_force": 10,
  "right_force": 10,
  "temp": 40,
  "vibration": 2
}
```

ESP32 每 500ms 请求一次，执行后上报状态。

### ESP32 → 云端（上报状态）

已有接口（患者 token 或设备 token）：

```
POST /api/v1/patients/me/device/telemetry
```

上报：`is_running`、`left_force`、`battery_level`、`hardware_status` 等。

## 六、常见问题

### Q1：部署后打开很慢？

免费 Render **休眠了**，第一次访问要唤醒，属正常现象。

### Q2：部署成功但登录后数据没了？

免费实例 **没有持久磁盘**，重启会重置 JSON 数据库。答辩前重新注册/用演示账号即可；要永久数据需 PostgreSQL。

### Q3：手机还能用局域网 IP 吗？

**不用了。** 云端部署后，测试设备只访问 `https://xxx.onrender.com`。

### Q4：Netlify 行不行？

Netlify 只能放 **静态网页**，**不能**跑你的 Express 后端。要 Netlify 得拆前后端，不如 Render 一条 Docker 简单。

### Q5：软硬件什么时候算「真正联调完成」？

至少满足：

- [ ] 云地址 24 小时可访问（或答辩前唤醒一次）  
- [ ] 测试设备浏览器能登录并完成理疗流程  
- [ ] ESP32 收到 START 后推杆真实动作  
- [ ] 急停 / 安全夹在 **固件层** 生效，不只是网页提示

## 八、本地 vs 云端对照

| | 本地 Mac | 云端 Render |
|--|----------|-------------|
| 启动 | `bash scripts/start-production.sh` | 自动运行，不用开 Mac |
| 地址 | `http://localhost:8080` | `https://xxx.onrender.com` |
| 测试设备 | 须同一 WiFi | 任何网络，有网即可 |
| 适合 | 开发调试 | 答辩、远程、测试机演示 |
| 硬件 | USB/串口调试 ESP32 | ESP32 必须 Wi-Fi 上云 |
