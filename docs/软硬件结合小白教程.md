# 膝悦 KneeJoy — 软硬件结合小白教程

> 适合：已经部署好 https://kneejoy.onrender.com，接下来要让**真实推杆/ESP32** 跟着软件动的人。  
> 不需要先懂很多代码，按顺序做就行。

---

## 先搞懂：三个东西各干什么

可以把它想成 **「遥控器 + 云记事本 + 真实机器」**：

| 名字 | 是什么 | 比喻 |
|------|--------|------|
| **手机网页** | 你在浏览器打开的 kneejoy.onrender.com | 遥控器：点「开始」「停止」 |
| **云服务器 Render** | 24 小时放在网上的程序 + 数据库 | 记事本：记下谁登录、处方、是否在理疗 |
| **ESP32 + 推杆** | 放在产品里的电路板 + 电机 | 真实机器：收到命令就推、就加热 |

**重要：** 云服务器**不能**用 USB 线直接推你的推杆。  
必须是：**手机点按钮 → 云记下命令 → ESP32 上网取命令 → 推杆动**。

---

## 整体流程（一张图记心里）

```
你在手机上点「开始理疗」
        ↓
云服务器：好，我记下了，参数是 左15N 右15N 温度42℃
        ↓
ESP32 每隔 1 秒问云：有新命令吗？
        ↓
云：有，START，参数 xxx
        ↓
ESP32：驱动推杆、加热模块
        ↓
ESP32 告诉云：我现在在跑，电量 88%
        ↓
手机网页刷新/轮询：显示「运行中」
```

# 第一阶段：让 ESP32 能上网（约 1 周）

### 你要买什么 / 准备什么

- ESP32 开发板（带 Wi-Fi 的那种）
- USB 数据线（连电脑）
- 电脑装 **Arduino IDE** 或 **PlatformIO**（二选一，Arduino 更简单）
- 知道家里/实验室 **Wi-Fi 名字和密码**

### 第一步：点亮板子

1. USB 插电脑  
2. Arduino IDE → 示例程序 → **Blink**（闪灯）  
3. 上传成功 = 板子没问题  

### 第二步：连 Wi-Fi

在程序里填：

```
WiFi 名称：你的WiFi名
WiFi 密码：你的WiFi密码
```

上传后，打开 **串口监视器**（115200 波特率），应看到类似：

```
WiFi 已连接
IP 地址：192.168.x.x
```

### 第三步：访问你的云网站

在 ESP32 程序里让它访问：

```
https://kneejoy.onrender.com/api/v1/health
```

串口应打印：

```
{"status":"ok","service":"KneeJoy API",...}
```

**做到这里 = 硬件已经能「看见」你的云软件了。**

# 第二阶段：面包板上让推杆动（约 1～2 周）

> 先不要管云，先证明：**发一条命令，推杆会动**。

### 你需要

- 推杆/直线电机 + 驱动模块（如 L298N、继电器模块等，按你们毕设选型）
- 杜邦线、面包板、电源（电压按电机规格，注意安全）

### 简单测试

1. ESP32 一个引脚 → 驱动模块 → 推杆  
2. 串口发 `START` → 推杆伸出  
3. 串口发 `STOP` → 推杆收回  

### 和软件对齐的「命令格式」

你们网页里已经在用类似这样的文字（日志里能看到）：

```
@STxCMD: L_F=15, R_F=15, TEMP=42, VIB=1
@STxSTOP
```

**毕设建议：** ESP32 固件就认这两种：

| 收到什么 | 做什么 |
|----------|--------|
| `@STxCMD: ...` | 按参数启动（左拉力、右拉力、温度、震动档） |
| `@STxSTOP` | 立刻停止，卸力 |

参数含义（和软件一致）：

| 参数 | 含义 | 例子 |
|------|------|------|
| L_F | 左侧拉力 | 15（单位 N，演示可先当档位） |
| R_F | 右侧拉力 | 15 |
| TEMP | 加热温度 | 42（℃） |
| VIB | 震动档位 | 0～3 |

# 第三阶段：云和设备「对上话」（约 1～2 周）

现在手机只和云说话，ESP32 也要和云说话。需要 **设备专用接口**（以后要加在 `server/` 里）。

### 为什么需要？

- 患者用手机 **账号密码** 登录  
- ESP32 **不能**用患者密码（不安全、也不方便）  
- 所以给每台设备一个 **设备 ID + 设备密钥**，像门禁卡

### 建议的简单做法（轮询，最适合小白）

**每台设备出厂写入：**

```
device_id = KJ-001
device_token = 一串随机密码（保密）
```

**ESP32 每隔 1 秒做两件事：**

**① 问云：有没有新命令？**

```
GET https://kneejoy.onrender.com/api/v1/device/commands?device_id=KJ-001
Header: Authorization: Bearer 设备token
```

云返回示例：

```json
{
  "command": "START",
  "left_force": 15,
  "right_force": 15,
  "temp": 42,
  "vibration": 1
}
```

或：

```json
{ "command": "NONE" }
```

**② 执行后告诉云：我现在啥状态**

```
POST https://kneejoy.onrender.com/api/v1/device/telemetry
Body: {
  "is_running": true,
  "left_force": 15,
  "battery_level": 88,
  "is_safety_clip_attached": true
}
```

### 和手机操作怎么连起来？

| 你在手机上点 | 云做什么 | ESP32 做什么 |
|--------------|----------|--------------|
| 连接设备 | 标记「已连接 Wi-Fi」 | 开始轮询 commands |
| 开始理疗 | 写入 START 命令到队列 | 取到 START → 推杆动 |
| 紧急停止 | 写入 STOP 命令 | 取到 STOP → 立刻停 |
| （自动） | 存训练记录 | 上报 telemetry |

# 第四阶段：安全（答辩必讲，不能省）

**这些必须在 ESP32 里做**，不能只在网页里做：

1. **安全夹未插紧** → 不允许 START，已在运行则 STOP  
2. **拉力超过上限**（如 35N）→ 不允许 START  
3. **和云断网超过 X 秒** → 自动 STOP（防止失控）  
4. **急停按钮**（硬件）→ 随时 STOP  

网页里已经有「安全栓未插紧无法启动」的提示，**真产品必须传感器接到 ESP32**。

# 第五阶段：网页里去掉「假连接」（开发同学做）

文件位置：`src/hardware/mockAdapter.ts`（现在是假的）

以后要改成：

1. 新建 `realDeviceAdapter.ts`（通过云和设备通信）  
2. 启动时替换 mock（`setHardwareAdapter(...)`）  
3. 点「开始理疗」时，确保云已经给设备队列里写了 START  

**你作为硬件同学**：只要保证 **云下发的 START/STOP，ESP32 能执行** 即可，网页会自动对齐。

# 常见问题（小白版）

### Q1：我已经部署了，为什么推杆还不动？

因为部署只把 **网页和云** 放上网了，**没有**把 ESP32 程序写进去。就像只做了遥控器 App，还没做接收信号的机器。

### Q2：ESP32 要连和电脑一样的 Wi-Fi 吗？

ESP32 要连 **能上网的 Wi-Fi**（实验室/家里路由器）。  
手机用 4G 打开网页也可以，不要求手机和 ESP32 同一网络。

### Q3：蓝牙还要吗？

可以后期加。毕设优先 **Wi-Fi + 云**，和「软件在 Render 上」最匹配。

### Q4：我不会写后端 API 怎么办？

把本文 **第三阶段** 的接口说明给写代码的同学，或让 Cursor/助手在 `server/routes/` 里加 `device.ts`。  
你先把 ESP32 **串口版 START/STOP** 做熟。

### Q5：数据会丢吗？

Render 免费版重启可能清空演示数据，答辩前重新登录演示账号即可。  
以后要长期存数据，再换 PostgreSQL 数据库。
