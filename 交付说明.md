# 膝悦 KneeJoy — 专业人员交付说明

> 版本：固件 v2.1.0 · API v1.0.0  
> 适用：接手本项目的嵌入式 / 全栈工程师

---

## 1. 系统架构

```
┌─────────────┐   HTTP/JSON    ┌──────────────┐   HTTP 轮询    ┌─────────────┐
│ Web UI      │ ─────────────► │ Express API  │ ◄────────────► │ ESP32 固件  │
│ React/Vite  │   Bearer Token │ port 3001    │  Device Token  │ 三模块执行  │
└─────────────┘                └──────────────┘                └─────────────┘
```

- **Web 端**：患者操作界面；Wi-Fi 连接为 mock（真实设备走 API 命令队列）。
- **API**：`server/routes/device.ts` 供 ESP32 轮询；`server/routes/patients.ts` 在 START 理疗时 `queueDeviceCommand`。
- **ESP32**：唯一真实硬件执行层（推杆 / 加热 / 震动）。

---

## 2. 硬件固件文件索引

路径：`firmware/esp32-kneejoy/KneeJoy_Device/`

| 文件 | 职责 |
|------|------|
| `actuator.cpp` | 推杆：拉力 N → PWM；支持 DEMO_LED / L298N |
| `heater.cpp` | 加热：温度 ℃ → PWM（开环） |
| `vibration.cpp` | 震动：档位 0~3 → PWM |
| `therapy_session.cpp` | START/STOP/SYNC、倒计时、安全联动 |
| `safety.cpp` | 安全夹 GPIO34、急停 GPIO35、拉力上限 |
| `cloud_client.cpp` | Wi-Fi、HTTP/HTTPS 轮询与 telemetry |
| `serial_cmd.cpp` | 串口协议 `@STxCMD` / `@STxSTOP` |
| `config.h` | **本地配置**（Wi-Fi、API IP、引脚、设备 Token） |
| `KneeJoy_Device.ino` | 主入口 |

## 4. API 契约（设备通道）

### GET `/api/v1/device/ping`

设备上线自检。

### GET `/api/v1/device/commands`

响应示例：

```json
{
  "command": "START",
  "left_force": 15,
  "right_force": 15,
  "temp": 42,
  "vibration": 1,
  "duration": 20,
  "max_force_limit": 35
}
```

`command` 枚举：`NONE` | `START` | `STOP` | `SYNC`

### POST `/api/v1/device/telemetry`

Body 字段与 `TherapyTelemetry` 一致；服务端写入 `devices[patient_id]`。

## 7. 引脚表（默认 DEMO_LED）

| GPIO | 功能 |
|------|------|
| 25 | 左推杆 PWM / LED |
| 26 | 右推杆 PWM / LED |
| 27 | 加热 PWM / LED |
| 14 | 震动 PWM / LED |
| 34 | 安全夹（LOW=已插紧） |
| 35 | 急停（LOW=按下） |
| 2 | 运行状态灯 |

L298N 模式见 `config.example.h` 中 `PROFILE_L298N` 段落。

## 8. 已知限制与扩展建议

| 项目 | 现状 | 建议 |
|------|------|------|
| 力反馈闭环 | 无 HX711 | 接 ADC/称重后扩展 `actuator.cpp` |
| 温控 | 开环 PWM | 接 DS18B20 做 PID |
| 电量 | 固定 95% | ADC 分压采样 |
| Web 蓝牙连接 | Mock | 可保留云端方案，不必做 Web BLE |
| HTTPS 本地 | 默认 HTTP | 生产部署时 `API_USE_TLS=1` |

## 10. 关键业务流

1. 患者登录 → `GET /patients/me/device`
2. 设置 Wi-Fi 连接 → `PATCH .../device/connection` → `queueDeviceCommand(SYNC)`
3. 开始理疗 → `POST .../treatment/sessions` → `queueDeviceCommand(START)`
4. ESP32 轮询取走 START → `therapy_session.start()` → 三模块输出
5. ESP32 `telemetry` 上报 → UI 轮询/刷新显示运行状态
